Lista de cosas a instalar y configurar para ejecutar el proyecto en local

1) Requisitos del sistema
- Git
  Descargar: https://git-scm.com/download/win
  
- Node.js 16.x+ (recomendado) y npm 8.x+
  Descargar: https://nodejs.org/ (usar LTS)
  O con winget: winget install -e --id OpenJS.NodeJS
  
- Python 3.11+
  Descargar: https://www.python.org/downloads/
  O con winget: winget install -e --id Python.Python.3.11
  
- MongoDB (servidor)
  Descargar: https://www.mongodb.com/try/download/community
  O con Docker (ver docker-compose.yml en este directorio)
  Opcionalmente MongoDB Compass: https://www.mongodb.com/products/tools/compass
  
- Neo4j (Desktop o servidor)
  Descargar: https://neo4j.com/download/
  O con Docker (ver docker-compose.yml en este directorio)
  Acceso: http://localhost:7474 (user: neo4j, pass: password)
  
- Ollama (si vas a usar `gemma3` LLM)
  Descargar: https://ollama.ai
  Después de instalar: ollama pull gemma3
  
- (Opcional) Docker y Docker Compose
  Descargar: https://www.docker.com/products/docker-desktop
  Windows: instalar desde Docker Desktop (incluye docker-compose)

2) Herramientas útiles
- nvm o nvm-windows (para manejar versiones de Node)
  Descargar: https://github.com/coreybutler/nvm-windows/releases
  
- Virtualenv (Python) o usar `python -m venv env`

- Windows: privilegios para ejecutar servicios o instalar servicios

3) Instalación automática de dependencias con PowerShell

Opción A: Script automático (RECOMENDADO)
  1. Abre PowerShell en la carpeta `Sistema Medico Frontend`
  2. Ejecuta:
     powershell -ExecutionPolicy Bypass -File .\install-dependencies.ps1
  3. El script:
     - Instala Git, Node.js, Python (si faltan)
     - Instala las dependencias npm de Frontend y Node API
     - Prepara el entorno virtual Python e instala dependencias
     - Te muestra qué instalar manualmente (MongoDB, Neo4j, Ollama)

Opción B: Con Docker (SIMPLIFICADO)
  Si tienes Docker Desktop instalado:
  1. Abre PowerShell en la carpeta `Sistema Medico Frontend`
  2. Ejecuta:
     docker-compose up -d
  3. Esto inicia MongoDB y Neo4j automáticamente
  4. Luego ejecuta el script de instalación sin Docker:
     powershell -ExecutionPolicy Bypass -File .\install-dependencies.ps1

Opción C: Manual (línea por línea)
  Ver la sección "3) Dependencias por componente" más abajo

4) Después de la instalación automática
- Node API (`Sistema Medico/node-api`):
  cd "Sistema Medico\node-api"
  npm install
  # revisar o crear .env (ya existe .env con DB_LINK, JWT_SECRET, APP_URL_CLIENT, APP_URL_API, PORT)
  npm run start:dev
  # seed opcional
  npm run seed

- React frontend (`Sistema Medico/material-react-app`):
  cd "Sistema Medico\material-react-app"
  npm install
  npm start

- FastAPI backend Python (`backend/backend`):
  cd "backend\backend"
  # activar venv si existe
  .\env\Scripts\activate
  pip install -r requirements.txt
  pip install py2neo ollama
  uvicorn main:app --port 8000

4) Servicios y puertos por defecto
- MongoDB: 27017
- Node API: 4000 (según .env en `node-api`)
- React: 3000
- FastAPI: 8000
- Neo4j Bolt: 7687

5) Variables de entorno importantes
- Node API (`node-api/.env`):
  DB_LINK=mongodb://localhost:27017
  JWT_SECRET=... (asegúrate de un secreto seguro)
  APP_URL_CLIENT=http://localhost:3000
  APP_URL_API=http://localhost:4000
  PORT=4000

- FastAPI / Neo4j (si usas `backend/backend/database/neo4j.py` env vars):
  NEO4J_URI=bolt://localhost:7687
  NEO4J_USER=neo4j
  NEO4J_PASS=password

- Ollama: asegúrate de que `ollama` esté instalado y el modelo `gemma3` descargado/activo si el backend lo requiere.

6) Instalación rápida con Docker (RECOMENDADO)
Si tienes Docker Desktop instalado, simplemente ejecuta desde la raíz `Sistema Medico Frontend`:
  docker-compose up -d

Esto inicia:
- MongoDB en puerto 27017
- Neo4j en puertos 7474 (HTTP) y 7687 (Bolt)

Después de esto, omite descargar MongoDB y Neo4j manualmente.

Para detener los contenedores:
  docker-compose down

Para ver los logs:
  docker-compose logs -f

7) Instalación MANUAL de MongoDB y Neo4j (sin Docker)
- MongoDB:
  1. Descargar desde https://www.mongodb.com/try/download/community
  2. Instalar (incluye la opción de MongoDB como servicio Windows)
  3. Verificar que se ejecuta en puerto 27017
  4. Opcionalmente usar MongoDB Compass (https://www.mongodb.com/products/tools/compass) para administración

- Neo4j:
  1. Descargar Neo4j Desktop desde https://neo4j.com/download/
  2. Instalar y ejecutar
  3. Crear una base de datos
  4. Verificar que Bolt esté en puerto 7687
  5. Acceder a http://localhost:7474 con user: neo4j, pass: password

8) Notas y comprobaciones

Antes de iniciar con deploy.ps1, asegúrate de:

✓ MongoDB está corriendo:
  - Con Docker: docker-compose up -d && docker ps (verás "mongodb" en la lista)
  - Manual: mongod ejecutándose o MongoDB service iniciado
  - Verificar: mongo --eval "db.stats()" o conectar con MongoDB Compass

✓ Neo4j está corriendo:
  - Con Docker: docker-compose up -d && docker ps (verás "neo4j" en la lista)
  - Manual: Neo4j Desktop abierto con instancia creada
  - Verificar: acceder a http://localhost:7474 (user: neo4j, pass: password)

✓ Node.js y npm están instalados:
  node --version
  npm --version

✓ Python está instalado:
  python --version
  pip --version

✓ Variables de entorno en `node-api/.env` son correctas:
  - DB_LINK=mongodb://localhost:27017 (o tu DB específica)
  - JWT_SECRET=tu_secreto_aqui
  - APP_URL_CLIENT=http://localhost:3000
  - APP_URL_API=http://localhost:4000
  - PORT=4000

✓ Si usas FastAPI con `ollama`:
  - ollama está instalado: ollama --version
  - Modelo descargado: ollama list (debería mostrar "gemma3")
  - O descargar: ollama pull gemma3

9) Iniciar todo después de la instalación

Opción A: Con el script deploy.ps1:
  powershell -ExecutionPolicy Bypass -File .\deploy.ps1

Opción B: Manualmente en ventanas separadas:
  Ventana 1 (Node API):
    cd "Sistema Medico\node-api"
    npm run start:dev

  Ventana 2 (React Frontend):
    cd "Sistema Medico\material-react-app"
    npm start

  Ventana 3 (FastAPI Python):
    cd "backend\backend"
    .\env\Scripts\activate
    uvicorn main:app --port 8000

10) URLs para acceder después de iniciar

  Frontend: http://localhost:3000
  Node API: http://localhost:4000
  FastAPI: http://localhost:8000
  MongoDB: localhost:27017 (conéctate con MongoDB Compass o mongo CLI)
  Neo4j: http://localhost:7474 (user: neo4j, pass: password)

11) Solucionar problemas comunes

Problema: "npm: command not found"
  Solución: Reinicia PowerShell o terminal después de instalar Node.js

Problema: "python: command not found"
  Solución: Verifica que Python esté en PATH. Reinstala si es necesario.

Problema: "Cannot find module" en Node API
  Solución: cd "Sistema Medico\node-api" && npm install

Problema: "ModuleNotFoundError" en FastAPI
  Solución: Activa venv (.\env\Scripts\activate) e instala: pip install -r requirements.txt

Problema: MongoDB connection refused
  Solución: Verifica que MongoDB esté corriendo. Si usas Docker: docker-compose up -d

Problema: Neo4j authentication failed
  Solución: Verifica que el password sea "password" o cambiar en .env (variable NEO4J_PASS)

Problema: Port already in use (3000, 4000, 8000, etc.)
  Solución: Encuentra el proceso que usa ese puerto:
    netstat -ano | findstr :3000
    taskkill /PID [PID] /F
